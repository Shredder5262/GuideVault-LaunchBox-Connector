# GitHub repository setup

Recommended repository name:

```text
GuideVault-LaunchBox-Connector
```

Recommended description:

```text
LaunchBox plugin for syncing and opening GuideVault manuals, strategy guides, and magazines.
```

Keep this plugin in its own repository. It has a different build dependency, release cadence, and install path than the GuideVault server.

## Create and push

From the extracted repository folder:

```powershell
git init
git branch -M main
git add .
git commit -m "Initial GuideVault LaunchBox Connector 0.4.22 release"
git remote add origin https://github.com/shredder5262/GuideVault-LaunchBox-Connector.git
git push -u origin main
```

Then create a GitHub release named:

```text
v0.4.22
```

Attach the binary release zip generated by:

```powershell
.\scripts\Package-PluginRelease.ps1 -LaunchBoxRoot "D:\HyperspinMasterbuild\LaunchBox"
```

## Important dependency note

The repository cannot build in a normal GitHub Actions runner unless the LaunchBox plugin API DLL is supplied during the workflow. Do not commit `Unbroken.LaunchBox.Plugins.dll` unless its license allows redistribution. The local build script uses the DLL from your installed LaunchBox folder.
